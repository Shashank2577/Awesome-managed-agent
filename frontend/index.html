How it works

A commander you can watch work.

One thread. One commander. Many ephemeral specialists. Every transition is an event. Every event is on screen.

Ingest the goal

You type a request. The commander acknowledges it and starts a thread with a budget.

Plan & hire

The commander publishes a plan DAG and hires the specialists by capability — many in parallel.

Pivot on evidence

If a specialist surfaces a clue that breaks the plan, the commander reverses course and hires more help.

Ship a report

A presenter agent composes a human-friendly evidence card with charts, findings, and next actions.

What you get

Streaming. Inspectable. Bounded.

Built on a strict execution constitution: events are the only communication primitive, agents are stateless, and every action is bounded by deterministic guardrails.

Live event stream

SSE feed of every plan, hire, transition, and pivot. Reconnect, dedupe by event id, replay from any sequence.

Inspectable plan DAG

Click any node to see attempts, output, and tool calls. The graph updates in real time as nodes transition.

Pivots on evidence

The commander reverses course mid-flight when a specialist surfaces something that breaks the plan.

Resumable threads

Checkpointed before fan-outs and pivots. Resume idempotently from the last valid sequence.

Human-friendly evidence

A presenter agent picks the right shape — bars, donuts, scorecards — to tell the story to a human.

Deterministic guardrails

Bounded by max-agents, max-parallel, max-time, max-cost, max-pivots. Every breach emits an event.

Architecture

Five layers. Clean separation.

Control, Execution, Event, State, and UI are independent. Plug in new agents, tools, and model providers without touching the core.
